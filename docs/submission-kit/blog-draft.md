# Five Agents, One Brain — Part 1: Laying the Foundation

*The first entry in a build journal we're writing as we go. Fittingly, the first draft of each post is written by Qwen itself, reading our own commit log and journal — then we edit. More on that at the end.*

We're building an agent society: five Qwen agents — `memory-echo`, `showrunner`, `git-committer`, `open-translate`, and `skippy-concierge` — that share a single memory brain and call Qwen Cloud at runtime. No central orchestrator. Each agent reads and writes the same persistent memory and does its own reasoning.

That's the vision. Right now, we've laid the foundation — and we're going to be honest about exactly how far that goes.

## Where we actually are

On **2026-05-26**, all five tracks began as scaffold: empty packages aligned to their agent names, nothing wired yet. On **2026-06-25** we merged into a single monorepo — `brain`, `agents`, `shared`, `infra`, `docs` — and added the root files every project needs: `LICENSE`, `.gitignore`, `.env.example`, and READMEs.

There are no deployed services. No demo videos. No runtime calls to Qwen Cloud yet. We're not going to pretend otherwise — the hard parts are still ahead of us.

## The hard bits (still ahead)

The interesting problems are exactly the ones we haven't solved yet:

- Wiring the shared memory layer in `brain/` so five independent agents can read and write one space.
- Routing inter-agent requests without a central orchestrator.
- Keeping memory consistent across concurrent Qwen invocations — what happens when two agents write the same key at once.

We'd rather show you these solved, with real commits behind them, than hand-wave them now.

## What's next

First, implement the shared memory layer in `brain/`. Then connect `memory-echo` to it. Only after that do the other agents start reading from and writing to the same space.
